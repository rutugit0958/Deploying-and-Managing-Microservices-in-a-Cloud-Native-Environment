# Jira Incident Management & AWS Cloud Support Project

##  Project Overview
This project demonstrates real-world Incident Management and Cloud Support operations using Jira and AWS.  
It simulates how support engineers handle infrastructure issues, track tickets, and resolve incidents following a structured workflow.

The project focuses on managing AWS-based incidents through a complete Jira ticket lifecycle.

---

##  Tools & Technologies Used
- Jira (Issue Tracking & Workflow Management)
- AWS EC2 (Cloud Infrastructure)
- Linux Server
- Apache / Application Logs
- Git & GitHub

---

##  Project Objectives
- Simulate real-world cloud support scenarios
- Manage incident lifecycle using Jira
- Troubleshoot AWS infrastructure issues
- Close tickets following proper workflow stages

---

##  Jira Workflow Used
To Do → In Progress → Waiting for Customer → Resolved

Each issue was tracked and transitioned step-by-step until final resolution.

---

##  Incidents Handled

###  EC2 Instance Not Reachable
- Diagnosed connectivity issue
- Verified security groups and instance status
- Restored service accessibility

###  Linux Server High CPU Usage
- Monitored system performance
- Identified resource-heavy processes
- Optimized and resolved CPU spike issue

###  Application Log Error Entries
- Analyzed server logs
- Identified root cause
- Fixed configuration/application issue
